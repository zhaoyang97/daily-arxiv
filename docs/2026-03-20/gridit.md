# GriDiT: Factorized Grid-Based Diffusion for Efficient Long Image Sequence Generation

**日期**: 2026-03-20  
**arXiv**: [2512.21276](https://arxiv.org/abs/2512.21276)  
**代码**: 无  
**领域**: 图像生成 / 视频理解  
**关键词**: image sequence generation, grid factorization, diffusion transformer, long-range consistency, medical imaging

## 一句话总结
提出 GriDiT 两阶段分解生成框架——Stage 1 将帧序列排成 K×K 网格图在低分辨率下建模帧间关联，Stage 2 逐帧超分重建——在 CT-RATE 医学影像上比 GenerateCT 快 3.4× 且 FVD 更优（998.43 vs 1092.3），支持 1024 帧长序列生成。

## 研究背景与动机

1. **领域现状**: 图像序列生成（视频、3D 体积切片、医学CT）的现有方法将帧作为大张量处理，面临严重的效率瓶颈和长程一致性挑战。

2. **现有痛点**: (a) 计算成本随帧数线性到超线性增长；(b) 固定序列长度，难以扩展到任意长度；(c) 需要领域特定先验，无法跨域迁移。

3. **核心 idea**: 将长序列分解为"网格概览→逐帧精修"两阶段——用空间换时间，将时序问题转化为二维布局问题。

## 方法详解

### 关键设计

1. **Stage 1 — 网格粗生成**:
   - 将 N 帧序列排成 K×K 网格（每帧下采样为 grid 中的一个子图像）
   - 3D 位置编码（行/列/帧索引）建模帧间关联
   - DiT 在网格图像上做扩散生成

2. **Stage 2 — 逐帧超分**:
   - 条件 DiT 将每个低分辨率格子超分为高分辨率帧
   - 各帧可并行处理

3. **Grid-based 自回归采样**:
   - 网格间用扩散 inpainting 衔接
   - 支持任意长度序列生成

## 实验关键数据

### 主实验

| 数据集 | 方法 | FVD ↓ | 推理时间 |
|--------|------|-------|---------|
| CT-RATE (512×512, 201 slices) | GenerateCT | 1092.3 | 184s |
| CT-RATE | **GriDiT** | **998.43** | **53.8s (3.4×)** |
| Minecraft (1024 frames) | Diffusion Forcing | 261.23 | - |
| Minecraft | **GriDiT** | **243.21** | - |

### 消融

| 配置 | FVD-128 |
|------|---------|
| 2D 位置编码 | 152.6 |
| **3D 位置编码** | **133.2** |

### 关键发现
- 训练 ≤400 帧但可泛化到 1024 帧——网格自回归策略实现了长度外推
- K=4 是最优网格大小（K=8 更快但牺牲质量）
- 跨域无需修改：医学CT/天空延时/Minecraft 一套架构通吃

## 亮点与洞察
- **将时序分解为空间**的思路巧妙：利用 2D DiT 的成熟能力处理本质上的 1D 序列问题
- 3D 位置编码让网格中的位置既编码空间且编码时序——使模型真正理解帧间关系
- 3.4× 加速的同时质量更优——分解策略避免了对整个序列的联合建模

## 局限性 / 可改进方向
- 两阶段分解引入了信息瓶颈——低分辨率网格可能丢失关键细节
- 自回归衔接点处的连续性可能出现伪影
- 仅评估了低分辨率（128-512），高分辨率视频生成的适用性待验证
