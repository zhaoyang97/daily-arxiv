# MORE-R1: Guiding LVLM for Multimodal Object-Entity Relation Extraction via Stepwise Reasoning with RL

**日期**: 2026-03-10  
**arXiv**: [2603.09478](https://arxiv.org/abs/2603.09478)  
**代码**: 有  
**领域**: 多模态/VLM / 信息抽取  
**关键词**: reinforcement learning, LVLM, relation extraction, chain-of-thought, GRPO

## 一句话总结
将多模态对象-实体关系抽取（MORE）形式化为带显式推理过程的生成任务，通过两阶段训练（SFT 冷启动 + GRPO 强化学习）和渐进采样混合策略，显著提升大视觉语言模型在关系抽取上的推理能力。

## 研究背景与动机

1. **领域现状**: MORE 要求从图文对中抽取对象-实体关系，分类式方法难以处理复杂关系，生成式方法缺乏推理能力。
2. **现有痛点**: (i) 现有方法可扩展性差，无法处理新增关系类型；(ii) 中间推理过程黑盒，难以解释。
3. **核心 idea**: 让 LVLM 学会逐步推理（识别实体 → 定位对象 → 分析关系），用 GRPO 在线优化推理质量。

## 方法详解

### 两阶段训练
1. **SFT 冷启动**: GPT-4o 生成高质量推理链数据，教 LVLM 基础推理模式
2. **GRPO 优化**: 基于规则的奖励函数（格式+长度+答案），渐进采样策略

### 渐进采样混合策略
- 将样本按难度分为易/难两组
- 早期 1:1 混合，后期逐步增加难样本比例
- 防止模型早期发散

## 实验关键数据
- MORE benchmark 上达到 SOTA
- 首次将 LVLM 成功应用于 MORE 任务
- GRPO 训练效率优于 PPO
- 显式推理链提供可解释性

## 亮点与洞察
- **渐进采样策略** 巧妙平衡训练稳定性与学习难度
- 规则奖励函数比 LLM 评分更可控
- 推理链设计使模型输出可追溯验证

## 局限性 / 可改进方向
- 依赖 GPT-4o 生成推理数据
- 推理步骤数固定，难以处理变长推理
