# RubiCap: Rubric-Guided Reinforcement Learning for Dense Image Captioning

**日期**: 2026-03-10  
**arXiv**: [2603.09160](https://arxiv.org/abs/2603.09160)  
**代码**: 有  
**领域**: 图像生成 / 密集字幕  
**关键词**: dense captioning, reinforcement learning, rubric-guided, GRPO, VLM

## 一句话总结
通过动态合成图像特定的评估准则（rubrics）并用其指导强化学习（GRPO），解决密集图像字幕中的验证难题——使 7B 模型在 CapArena 上以 70.8% 胜率超越 GPT-4V 生成，甚至超越人类专家标注质量。

## 研究背景与动机

1. **领域现状**: 密集图像字幕对视觉-语言预训练和文生图模型至关重要，但高质量标注极其昂贵。
2. **现有痛点**: (i) SFT 导致语言多样性坍塌；(ii) RL 奖励要么基于 NLP 指标（过度简化），要么基于 VLM 评分（不透明且易被 hack）。
3. **核心 idea**: 用 LLM 自动生成细粒度的二元可检验准则（rubrics），将整体质量分解为多维度评分。

## 方法详解

### 自动准则合成（3 步）
1. **提取共识**: 教师模型委员会（多个 VLM）生成描述，提取多数意见
2. **诊断缺陷**: 对比学生输出与教师共识，识别具体差异
3. **形式化规则**: 为每个缺陷设计二元可检验准则，附权重表示严重度

### 准则引导的 GRPO
- 复合奖励 = 格式 + 长度 + 各准则项得分 / 总权重
- 准则是**样本特定**的（非固定模板），适应当前图像内容和失败模式
- 多维评估避免单一评分的奖励 hacking

## 实验关键数据
- CapArena 7B 模型 70.8% 胜率 vs 基线 50%
- 超越 GPT-4V 输出和人类专家标注
- 盲目排名中幻觉率最低
- 3B 模型在固定字数下匹配 32B 基线
- 用 RubiCap 生成的数据做 VLM 预训练，效果优于 GPT-4V 数据

## 亮点与洞察
- **动态准则合成**远优于静态检查表——每张图的评估标准都不同
- 多维评估有效防止奖励 hacking
- **RL 比 SFT 更好保留预训练能力**——不会导致语言多样性坍塌
- 小模型生成的高质量字幕可反哺大模型预训练

## 局限性 / 可改进方向
- 依赖多个教师模型（API 成本高）
- 准则合成过程本身可能有偏见
