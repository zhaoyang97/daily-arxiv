# Test-time Ego-Exo-centric Adaptation for Action Anticipation

**日期**: 2026-03-10  
**arXiv**: [2603.09798](https://arxiv.org/abs/2603.09798)  
**代码**: 有  
**领域**: 机器人 / 跨视角适应  
**关键词**: test-time adaptation, ego-exocentric, action anticipation, multi-label, prototype

## 一句话总结
首次定义测试时自我-异我视角适应任务，提出多标签原型增长（ML-PGM）和双线索一致性（DCCM）模块，使源视角训练的模型在测试时无需目标视角数据即可在线自适应到新视角，在 EgoMe-anti 和 EgoExoLearn 基准上超越 SOTA。

## 研究背景与动机

1. **领域现状**: 自我视角（Ego）和异我视角（Exo）的高效适应对人机协作等应用至关重要。
2. **现有痛点**: (i) 跨视角适应通常需要目标视角训练数据；(ii) 动作预测中多动作候选使单标签假设失效；(iii) 视角间存在显著的时空差距。
3. **核心 idea**: 用多标签学习处理多动作候选，用视觉+文本双线索弥合视角差异。

## 方法详解

### 多标签原型增长模块（ML-PGM）
- 为每个样本分配多个伪标签（非单标签）
- 构建类别级记忆库，熵优先队列筛选高置信表示
- 基于置信度加权计算原型

### 双线索一致性模块（DCCM）
- 轻量叙述生成器产生文本线索（动词+名词描述动作进展）
- 视觉线索提供空间对象信息
- KL 散度约束两模态预测一致性

## 实验关键数据
- EgoMe-anti: Exo2Ego 准确率 79.03%
- EgoExoLearn: 超越基线 15%+
- 消融验证每个组件贡献 ≥5%
- 超越 ML-TTA 基线 6-10 个百分点

## 亮点与洞察
- **首创测试时跨视角适应任务**——任务定义本身是重要贡献
- 多标签假设优雅处理动作预测的多义性
- 文本线索补充单帧视觉缺失的时间信息
- 三组件（原型+视觉+文本）互补融合

## 局限性 / 可改进方向
- 依赖预训练 CLIP 模型
- 叙述生成可能引入额外误差
- 对长期时间依赖支持有限
