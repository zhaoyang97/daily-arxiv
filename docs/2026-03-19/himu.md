# HiMu: Hierarchical Multimodal Frame Selection for Long Video Question Answering

**日期**: 2026-03-19  
**arXiv**: [2603.18558](https://arxiv.org/abs/2603.18558)  
**领域**: 视频理解 / 多模态/VLM  
**关键词**: 长视频QA, 帧选择, 神经符号推理, 模糊逻辑, 无训练

## 一句话总结
提出 HiMu，用单次 LLM 调用将查询分解为层次逻辑树，叶节点路由到轻量多模态专家（CLIP/OCR/ASR/CLAP），通过模糊逻辑算子自下而上组合产生帧满意度曲线——在 Qwen3-VL 8B + 16 帧配置下超越所有帧选择器，在 GPT-4o 下以 10× 更少 FLOPs 超越 Agent 方法。

## 研究背景与动机

1. **领域现状**: 长视频 QA 受限于 LVLM 上下文窗口——帧选择至关重要。现有方法分两极：相似度选择器（快但无法处理组合查询）和 Agent 方法（准但成本 10-100× 更高）。

2. **核心矛盾**: 组合推理（"旁白提到化学反应后，左边烧杯发生了什么？"）需要同时推理音频（旁白）和视觉（烧杯）——单一嵌入不可能做到。但 Agent 的迭代 LVLM 推理太贵。

3. **核心 idea**: 不需要 LVLM 做帧选择！将查询分解为逻辑树 → 轻量专家逐帧评估（CLIP检视觉/ASR检音频/OCR检文字）→ 模糊逻辑组合 → 生成帧满意度曲线选最优帧。

## 方法亮点

- **无训练即插即用**: 无需额外训练，可直接接入任何 LVLM
- **多模态专家路由**: 查询中的视觉部分→CLIP，音频部分→ASR/CLAP，文字→OCR——自动路由
- **时序推理**: 模糊逻辑的时序算子强制"A之后B"的顺序关系和邻接关系
- **10× FLOPs 减少**: 缓存视频专家特征后，每次查询只需轻量树评估

## 实验关键数据

| 配置 | Video-MME | 计算量 |
|------|-----------|--------|
| HiMu + Qwen3-VL-8B (16帧) | 超越所有选择器 | 低 |
| HiMu + GPT-4o | 超越 Agent 方法 | Agent 的 1/10 |
| 相似度选择器 | 平台期 | 最低 |
| Agent 方法 (32-512帧) | 高准确率但成本高 | 10-100× |

## 亮点与洞察

- **"组合推理不需要 LVLM"**: 帧选择阶段的推理可以完全由轻量专家+逻辑组合完成——LVLM 只需在最终选出的少量帧上做回答。
- **神经符号方法在视频理解中的成功案例**: 用逻辑树+模糊逻辑做可解释的时序推理——既高效又可解释。
