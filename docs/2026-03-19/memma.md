# MemMA: Coordinating the Memory Cycle through Multi-Agent Reasoning and In-Situ Self-Evolution

**日期**: 2026-03-19  
**arXiv**: [2603.18718](https://arxiv.org/abs/2603.18718)  
**代码**: [GitHub](https://github.com/ventr1c/memma)  
**领域**: LLM Agent  
**关键词**: 记忆增强LLM, 多智能体, 记忆循环, 自演化修复, 长程交互

## 一句话总结
提出 MemMA，一个即插即用的多智能体框架，通过前向路径（Meta-Thinker 指导记忆构建和迭代检索）和后向路径（原位自演化记忆修复）协调记忆循环的三个阶段（构建-检索-利用），在 LoCoMo 上一致超越现有记忆增强基线。

## 研究背景与动机

1. **领域现状**: 记忆增强 LLM Agent 维护外部记忆库支持长程交互。现有系统将记忆构建、检索和利用视为独立子程序。

2. **现有痛点**: (a) **前向路径的策略盲区**——构建和检索由局部启发式驱动，缺乏显式的战略推理：构建时近视地累积冲突事实，检索时盲目重复搜索不缩小信息差距；(b) **后向路径的稀疏延迟反馈**——下游答错很难归因到哪次构建决策导致的错误，记忆中的遗漏和冲突持续积累。

3. **核心 idea**: 将记忆循环视为闭环——前向由 Meta-Thinker 做战略规划指导 Memory Manager（构建）和 Query Reasoner（检索）；后向通过合成探测 QA→验证记忆→失败转化为即时修复动作。

## 方法亮点

- **Planner-Worker 架构**：Meta-Thinker 产生结构化指导→ Memory Manager 执行记忆操作 / Query Reasoner 执行诊断式迭代检索
- **原位自演化**：每个 session 后自动生成探测问题、验证当前记忆、将失败转为修复动作——不等下游出错再反馈
- **即插即用**：可叠加到 3 种不同存储后端上均有提升
- **跨 backbone**：在多个 LLM backbone 上一致有效

## 实验关键数据

- 在 LoCoMo 基准上一致超越现有记忆增强基线
- 作为插件提升 3 种不同存储后端的性能
- 跨多个 LLM backbone 通用
