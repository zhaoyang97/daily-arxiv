# Multi-Modal Character Localization and Extraction for Chinese Text Recognition

**日期**: 2026-03-14  
**arXiv**: [2603.13886](https://arxiv.org/abs/2603.13886)  
**代码**: [LER](https://github.com/Pandarenlql/LER)  
**领域**: 多模态VLM / 文字识别  
**关键词**: Chinese text recognition, CLIP, character decoupling, radical decomposition, parallel decoding

## 一句话总结
提出 LER（Localization-Extraction-Recognition）框架，通过 CLIP 多模态信息辅助字符定位 + 显式字符特征解耦 + 部首感知 IDS 解码器，解决中文场景文字识别中的误差累积和注意力漂移问题，在 CTR benchmark 上达到 SOTA。

## 研究背景与动机

1. **领域现状**: STR 方法在英文上表现优秀，但直接应用到中文准确率骤降（ABINet 在中文场景仅 70.28%）。中文有 27000+ 字符类别和复杂内部结构（偏旁部首+空间结构）。

2. **现有痛点**: (a) 自回归解码器有**误差累积**问题 — 一个字符错误导致后续连锁错误。(b) 并行解码器的位置查询相似度高导致**注意力漂移**。(c) 语言模型后处理可能**错误纠正**图像中的实际文本。

3. **核心 idea**: 显式解耦每个字符特征后独立识别，从根本上避免误差累积和自动纠错；用 CLIP 的图文对齐能力增强字符定位，用 IDS 部首解码器增强解耦质量。

## 方法详解

### 整体框架
CNN 编码器 → 多模态定位模块（CLIP 文本特征+级联注意力定位字符位置）→ 提取模块（char cutter 显式解耦字符特征）→ 识别模块（字符分类器 + IDS 部首解码器辅助训练）。

### 关键设计

1. **多模态定位模块（MLB）**: N 级级联结构，第一级用 CLIP 文本特征（"a character of Chinese" 等内容无关 prompt 的编码聚合）作为初始查询，后续级使用前一级预测嵌入+语言信息。Masked Multi-head Self-Attention 利用语言上下文辅助精确定位。

2. **字符提取模块（Char Cutter）**: 定义字符区域大小 $(c_h, c_w)$，用 char prompt 通过 M 层 cross-attention 从定位特征中提取独立字符特征 $F \in R^{L \times c_h \times c_w \times D}$。字符特征独立后续处理，从根本避免误差累积。

3. **IDS 部首解码器（训练时使用）**: 预测字符的部首序列（10 种空间结构 + 562 种部首），帮助提取模块学得更完整的字符特征（只有包含完整部首才能正确解码 IDS）。推理时丢弃以提升效率。

### 训练策略
两阶段：Stage 1 训练编码器+定位模块（$L_{loc}$）；Stage 2 联合训练所有模块（$L = L_{char} + L_{ids} + L_{loc}$）

## 实验关键数据

### 主实验（CTR Benchmark，LACC/NED）

| 方法 | 场景 | 网页 | 文档 | 手写 | 平均 | Params |
|------|------|------|------|------|------|--------|
| ABINet | 64.65 | 66.55 | 97.40 | 52.50 | 70.28 | 36.7M |
| SMTR | - | - | - | - | 78.38 | 27.4M |
| **LER** | **-** | **-** | **-** | **-** | **79.45** | **15.5M** |

### 关键发现
- 平均提升 1.07%，但在最难的手写场景提升 2.4%（密集文本+多样书写风格）
- 参数量仅 15.5M，远小于 ABINet (36.7M) 和 SMTR (27.4M)
- 在英文 Union14M benchmark 上同样表现优秀，证明框架通用性
- NED 指标比 LACC 更能反映误差累积程度

## 亮点与洞察
- **CLIP 辅助定位**是巧妙的跨模态知识迁移 — 用 CLIP 的图文对齐能力帮助定位字符区域，比纯位置编码查询更精确
- **IDS 解码器仅训练时使用**的设计很实用 — 部首结构约束提升了 char cutter 的提取质量，但推理时无额外开销
- **模型极轻量**（15.5M）却超越大数倍的模型，说明显式字符解耦的架构设计比堆更多参数更有效

## 局限性 / 可改进方向
- 最大序列长度 L=25，超长文本行可能截断
- CLIP 的文本 prompt 设计目前是启发式的，可能还有优化空间
- IDS 部首解码器信息未在推理时利用，可能有潜力做部首级别的 ensemble
