# Garments2Look: A Multi-Reference Dataset for High-Fidelity Outfit-Level Virtual Try-On

**日期**: 2026-03-14  
**arXiv**: [2603.14153](https://arxiv.org/abs/2603.14153)  
**代码**: [Garments2Look](https://github.com/ArtmeScienceLab/Garments2Look)  
**领域**: 图像生成 / 虚拟试穿  
**关键词**: virtual try-on, outfit-level, multi-reference, dataset, fashion AI

## 一句话总结
构建 Garments2Look outfit 级虚拟试穿数据集——80,041 对穿搭级样本，涵盖 40+ 大类 300+ 子类，每对提供 3-12 张参考图，支持 1-5 层叠穿和 5 种造型技巧，通过 LLM 驱动的穿搭知识库 + 合成 pipeline + 严格筛选构建。

## 研究背景与动机

1. **领域现状**: 虚拟试穿已从单件服装推进到较好效果，但真实时尚以完整穿搭为中心，涉及多件单品叠穿、配饰搭配、造型技巧。

2. **现有痛点**: 现有 VTON 数据集类别有限、缺乏 outfit 多样性和正确标注、不支持叠穿推理。SOTA VTON 方法在完整穿搭组合上表现很差。

3. **核心 idea**: 构建首个 outfit 级多参考 VTON 数据集，通过时尚专家验证的穿搭知识库 + LLM 驱动生成 + 严格质检。

## 方法详解

### 数据合成 Pipeline
1. **穿搭知识库**: 65 种时尚风格，专家验证的叠穿规则和搭配约束
2. **LLM 驱动生成**: 生成用户情境+穿搭清单，含叠穿顺序约束
3. **商品检索**: 重加权采样确保类别均匀分布
4. **Look 合成**: 用 Nano Banana/Gemini-2.5-Flash 生成最终穿搭图
5. **严格筛选**: 规则+专家审核，仅保留约 40% 合格图像

## 实验关键数据

| 统计 | 数值 |
|------|------|
| 穿搭对数 | 80,041 |
| 大类/子类 | 40+ / 300+ |
| 平均参考图/对 | 4.48 (3-12) |
| 叠穿层数 | 1-5 层 |
| 造型技巧 | 5 种 |
| 分辨率 | ~1024×1024 |

### 关键发现
- 现有 SOTA VTON 方法在完整穿搭上挑战很大，尤其叠穿推理和造型技巧
- 数据集的 40% 保留率体现了严格质量控制

## 亮点与洞察
- **从单件试穿到穿搭级别**是数据集层面的重要升级 — 更接近真实时尚需求
- **知识驱动的穿搭规则**避免了随机搭配导致的不合理组合
