# WeEdit: A Dataset, Benchmark and Glyph-Guided Framework for Text-centric Image Editing

**日期**: 2026-03-12  
**arXiv**: [2603.11593](https://arxiv.org/abs/2603.11593)  
**代码**: 无  
**领域**: 图像生成 / 文本编辑  
**关键词**: text-centric editing, glyph-guided, multilingual, reinforcement learning, image editing

## 一句话总结
提出 WeEdit，一个面向图像中文字修改/翻译/重排的系统方案——基于 HTML 的自动数据生成 pipeline 构建 330K 训练对（覆盖 15 种语言）+ 字形引导微调注入空间内容先验 + 多目标强化学习对齐指令遵循/文字清晰度/背景保持，在多语文字编辑上大幅超越现有开源模型。

## 研究背景与动机

1. **领域现状**: 指令驱动图像编辑已在物体/风格操作上取得进展，但文字编辑（修改图中文字内容/翻译/重排）仍困难——现有模型常产生模糊或幻觉字符。

2. **现有痛点**: 缺少针对文字编辑的专用训练范式、大规模数据集和标准化 benchmark。

3. **核心 idea**: 构建闭环的训练-评估系统：大规模数据 + 分阶段训练（字形引导 SFT + 多目标 RL）。

## 方法详解

### 数据构建
- 基于 HTML 的自动编辑 pipeline，生成 330K 训练对
- 覆盖多样编辑操作和 15 种语言
- 提供双语和多语 benchmark

### 训练策略
1. **Stage 1: 字形引导监督微调** — 注入文字的空间位置和内容先验
2. **Stage 2: 多目标强化学习** — 同时优化指令遵循、文字清晰度、背景保持

## 实验关键数据

| 评估维度 | 关键结果 |
|---------|---------|
| 文字清晰度 | 大幅超越现有开源模型 |
| 多语支持 | 15 种语言覆盖 |
| 背景保持 | 多目标 RL 有效平衡编辑与保持 |

## 亮点与洞察
- HTML 驱动的数据生成 pipeline 可扩展性强，15 种语言的覆盖很实用
- 两阶段训练（SFT + RL）的设计思路可迁移到其他精细编辑任务
- 首个系统性的多语文字编辑方案

## 局限性 / 可改进方向
- 仅凭摘要无法获得具体定量数据
- 字形先验对手写/艺术字体的泛化需验证
