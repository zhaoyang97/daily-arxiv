# UniCompress: Token Compression for Unified Vision-Language Understanding and Generation

**日期**: 2026-03-11  
**arXiv**: [2603.11320](https://arxiv.org/abs/2603.11320)  
**代码**: 无  
**领域**: 多模态VLM / 模型压缩  
**关键词**: token compression, unified model, visual tokenizer, meta tokens, plug-in

## 一句话总结
提出 UniCompress，面向统一视觉-语言模型的即插即用视觉 token 压缩框架，通过可学习全局元 token 引导的压缩-解压机制将视觉 token 压缩 4 倍，理解任务掉点 ≤3%，生成 FID 增加 ≤5，推理延迟降低 41.8%。

## 研究背景与动机

1. **领域现状**: 统一模型（UniTok、Vila-U、VARGPT 等）将图像编码为离散 token 与文本一起处理，但 512×512 图像需要 1024 个 token，序列很长。

2. **现有痛点**: 朴素的 token 下采样或均匀剪枝虽对理解有效，但对生成性能损害超过 15%——生成依赖精细空间一致的 token 来重建细节。

3. **核心矛盾**: 理解和生成对 token 粒度需求不同——理解可容忍压缩，生成对信息丢失极敏感。需要在统一框架下同时满足两者。

4. **核心 idea**: 引入全局元 token 作为语义锚点，压缩时保留局部显著证据 + 全局语义，解压时用全局 token 引导恢复长程结构和局部纹理。

## 方法详解

### 整体框架
两阶段训练：Stage 1 冻结 LLM 训练 tokenizer（压缩器+解压器+全局提取器）；Stage 2 冻结 tokenizer 在压缩 token 上微调 LLM。

### 关键设计

1. **全局元 token 提取器**: 用可学习查询从密集 token 序列提取 $N_g$ 个全局语义 token $\mathbf{G}$，捕获场景级信息
2. **平均池化压缩器**: 用固定的 $s \times s$ 平均池化将 $H \times W$ token 压缩为 $H/s \times W/s$，简单高效
3. **全局引导解压器**: 条件于全局 token 自回归恢复密集 token，全局 token 作为语义锚点修复细节损失
4. **两阶段训练**: 先训 tokenizer 堆栈学压缩-重建，再冻结 tokenizer 训 LLM 适应压缩 token

## 实验关键数据

### 主实验 — 理解基准

| 方法 | GQA | MME | POPE | Seed-bench |
|------|-----|-----|------|-----------|
| VARGPT | 58.12 | 1290.65 | 88.04 | 50.54 |
| VARGPT-Compressed | 55.90 | 1272.80 | 84.99 | 48.41 |
| BAGEL | 60.05 | 1312.40 | 89.20 | 51.10 |
| BAGEL-Compressed | 59.10 | 1304.10 | 88.60 | 50.80 |

### 生成基准

| 方法 | FID↓ | CLIP↑ |
|------|------|-------|
| VARGPT | 14.77 | 24.2 |
| VARGPT-Compressed | 15.02 | 21.6 |
| BAGEL | 12.73 | 32.0 |
| BAGEL-Compressed | 17.22 | 28.8 |

### 关键发现
- 4 倍压缩下理解掉点 ≤3%，但生成退化因模型而异（BAGEL FID 12.73→17.22）
- 推理延迟降低高达 41.8%，训练时间减少 15.4%
- 全局元 token 是生成质量保持的关键——无全局 token 时生成严重退化
- 即插即用设计：不需重训整个 LLM

## 亮点与洞察
- **理解-生成的非对称压缩**: 清晰揭示了两个任务对 token 粒度的不同需求
- **全局元 token 设计**: 优雅地弥合压缩导致的全局信息损失，对生成尤为关键

## 局限性 / 可改进方向
- 生成 FID 增加对某些模型仍较大（OpenUni: 16.45→24.29），压缩比4×可能过激
- 仅验证固定压缩率，内容自适应压缩（论文提及但未实验）可能更优
