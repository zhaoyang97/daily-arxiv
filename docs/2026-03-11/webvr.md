# WebVR: Benchmarking Multimodal LLMs for WebPage Recreation from Videos via Human-Aligned Visual Rubrics

**日期**: 2026-03-11  
**arXiv**: [2603.13391](https://arxiv.org/abs/2603.13391)  
**代码**: 有（Homepage/GitHub/HuggingFace）  
**领域**: 多模态VLM / 代码生成  
**关键词**: video-to-webpage, MLLM benchmark, visual rubrics, code generation, web development

## 一句话总结
提出 WebVR，首个视频到网页生成基准——175 个合成网页 + 细粒度人对齐视觉评分体系，评估 19 个 MLLM 从演示视频忠实重建网页的能力，自动评估与人类偏好达 96% 一致率。

## 研究背景与动机

1. **领域现状**: MLLM 正快速发展端到端从视觉输入生成可执行代码的能力。Web 开发中设计师常用屏幕录像而非完整设计文档传达需求。

2. **现有痛点**: 现有网页生成基准仅从文本或静态截图输入，无法评估动态行为（过渡动画、交互时序）；评估标准粗粒度，缺乏结构化视觉保真度评分。

3. **核心矛盾**: 视频天然携带交互流、过渡时序、动画连续性等丰富信号，但视频条件下的网页生成完全没有标准化评测。

4. **核心 idea**: 构建控制合成 pipeline 生成 175 个多类别网页（非爬取），设计 5 维细粒度评分体系（布局/色彩排版/组件完整性/动画/交互），用 MLLM-as-judge 自动评分。

## 方法详解

### 整体框架（4 阶段数据合成管线）
1. **种子数据准备**: 收集真实网页录屏 → MLLM 结构化描述 → **语义重主题化**（保留布局/交互逻辑但替换所有语义内容→防数据污染）→ 分类到 8 个类别
2. **视觉资产检索**: LLM 生成搜索关键词 → Unsplash API 检索图像 → 固定顺序提供给模型
3. **候选生成+执行**: 5 个代码生成模型各生成一个 HTML 实现 → 标准化沙箱 Chromium（2560×1440, 30fps）渲染+录屏
4. **自动过滤+精炼**: 基于 rubric 的 MLLM 评分选择最佳候选 → 每模型 top-50 → 过滤 S<0.5 → **175 个基准实例**

### 评估协议（核心设计）
- **任务**: 给定参考视频 $V_{ref}$ + 视觉资产 $A$ → MLLM 生成独立 HTML → 沙箱执行+录屏 $V_{gen}$ → 与 $V_{ref}$ 对比
- **4 维 Visual Rubric**: Global Aesthetics（整体色调/排版）、Navigation & Footer（持久UI组件）、Section-Specific Layouts（分区布局/对齐/间距）、Interaction & Motion（悬停/滚动/动画/状态转换）
- **原子化二元准则**: 每个 $r_k$ 只检查一个属性（存在性/对齐/颜色/动画触发等），每实例 32-248 条准则
- **评分**: $S = \frac{1}{K}\sum_{k=1}^K \hat{y}_k$，单次 MLLM 推理判断所有准则满足状态
- **人类对齐**: 与人类偏好 **96% 一致率**

### 关键设计动机
- **为什么用视频而非截图？** 截图无法评估动画时序、滚动行为、悬停反馈等动态交互
- **为什么合成而非爬取？** 防止模型在预训练中见过相同网页→数据污染
- **为什么提供视觉资产？** 让模型专注布局和交互重建，而非图像搜索
- **为什么用原子化二元准则？** 粗粒度评分（如 1-10）太主观；二元 pass/fail 更可靠且可解释

## 实验关键数据

### 主实验（19 个模型，Kimi-K2.5 作评判）

| 模型 | GA | NF | SSL | IM | Overall |
|------|----|----|-----|-----|---------|
| GLM-4.6V | 22.8 | 15.4 | 7.2 | 14.4 | 11.4 |
| Qwen3-VL-30B-Thinking | 53.4 | 60.5 | 33.5 | 20.2 | 37.7 |
| Qwen3-VL-235B-Thinking | 61.2 | 68.0 | 43.1 | 29.3 | 46.8 |
| Claude 系列 | 高 | 高 | 高 | 低 | 中上 |
| GPT-5 系列 | **最高** | 中高 | **最高** | 中 | **领先** |

### 关键发现
- **Interaction & Motion 是所有模型的最大短板**: 即使最强模型 IM 维度也仅 ~30%，远低于 GA/NF 的 60%+
- **Thinking 模式显著提升**: Qwen3-VL-30B Thinking vs Non-thinking: 37.7% vs 21.4%（+16.3%）
- **闭源 > 开源差距大**: 整体最高 ~55% vs 开源最高 ~47%
- **96% 人类偏好一致率**——自动评估可靠

## 亮点与洞察
- **首个视频→网页基准**: 填补了动态交互生成评估的空白——之前所有基准仅考虑静态截图或文本
- **语义重主题化防污染** 的设计很聪明——保留结构但替换语义=既真实又安全
- **原子化 rubric 比整体评分信息量大得多**——可以定位"到底哪个按钮的悬停没做对"

## 局限性 / 可改进方向
- 175 个网页规模偏小，复杂度分布可能不够均匀
- 评估依赖 MLLM-as-judge，可能对某些模型风格有偏好
- 仅测试单页面，多页面导航/路由未涉及

## 相关工作与启发
- **vs ArtifactsBench**: 用分阶段截图近似动态→稀疏时序采样无法捕捉连续动画；WebVR 直接用视频
- **vs WebUIBench**: 静态截图→代码评估；WebVR 扩展到视频输入+视频输出对比
