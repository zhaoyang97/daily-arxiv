# WebVR: Benchmarking Multimodal LLMs for WebPage Recreation from Videos via Human-Aligned Visual Rubrics

**日期**: 2026-03-11  
**arXiv**: [2603.13391](https://arxiv.org/abs/2603.13391)  
**代码**: 有（Homepage/GitHub/HuggingFace）  
**领域**: 多模态VLM / 代码生成  
**关键词**: video-to-webpage, MLLM benchmark, visual rubrics, code generation, web development

## 一句话总结
提出 WebVR，首个视频到网页生成基准——175 个合成网页 + 细粒度人对齐视觉评分体系，评估 19 个 MLLM 从演示视频忠实重建网页的能力，自动评估与人类偏好达 96% 一致率。

## 研究背景与动机

1. **领域现状**: MLLM 正快速发展端到端从视觉输入生成可执行代码的能力。Web 开发中设计师常用屏幕录像而非完整设计文档传达需求。

2. **现有痛点**: 现有网页生成基准仅从文本或静态截图输入，无法评估动态行为（过渡动画、交互时序）；评估标准粗粒度，缺乏结构化视觉保真度评分。

3. **核心矛盾**: 视频天然携带交互流、过渡时序、动画连续性等丰富信号，但视频条件下的网页生成完全没有标准化评测。

4. **核心 idea**: 构建控制合成 pipeline 生成 175 个多类别网页（非爬取），设计 5 维细粒度评分体系（布局/色彩排版/组件完整性/动画/交互），用 MLLM-as-judge 自动评分。

## 方法详解

### 基准构建
- 175 个网页，涵盖电商、作品集、落地页、娱乐、教育等类别
- 控制合成 pipeline 生成，确保不与现有网页重叠
- 每个任务配套视觉资产，让模型专注布局和交互重建

### 评估体系
- **5 维评分**: 布局结构、色彩排版、组件完整性、动画质量、交互正确性
- 生成代码在标准化沙箱中渲染和录制
- MLLM 评判器对比执行视频和参考视频打分
- 与人类偏好 96% 一致率

## 实验关键数据

### 主实验（19 个模型）

| 模型 | 布局 | 色彩 | 组件 | 动画 | 交互 | 总分 |
|------|------|------|------|------|------|------|
| GPT-5 系列 | 高 | 中高 | 高 | 中 | 中低 | 领先 |
| Claude 系列 | 高 | 中 | 高 | 低 | 低 | 中上 |
| 开源 MLLM | 中 | 中低 | 中 | 低 | 低 | 较弱 |

### 关键发现
- **精细样式和动画是最大短板**: 所有模型在动画质量和交互正确性上都明显弱于布局和组件
- 96% 人类偏好一致率——验证了自动评估的可靠性
- 视频输入比静态截图提供更多有用信号，但模型对时序信息利用不足

## 亮点与洞察
- **首个视频→网页基准**: 填补了动态交互生成评估的空白
- **人对齐评分体系**: 5 维评分比简单 pass/fail 信息量大得多

## 局限性 / 可改进方向
- 175 个网页规模偏小，复杂度分布可能不够均匀
- 合成网页可能不完全反映真实项目复杂度
- 评估依赖 MLLM-as-judge，可能对某些模型有偏好
